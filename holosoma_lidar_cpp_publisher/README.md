# Holosoma LiDAR C++ Publisher

ROS2 C++ publisher for MuJoCo LiDAR frames generated by `holosoma`.

The simulator still performs MuJoCo ray tracing, then sends raw `float32` XYZ
returns over a conflated low-watermark ZeroMQ stream. This node builds and
publishes either `sensor_msgs/msg/PointCloud2` or `livox_ros_driver2/msg/CustomMsg`
outside the simulator process.

Build inside a ROS2 environment:

```bash
sudo apt-get install -y libzmq3-dev python3-catkin-pkg python3-catkin-pkg-modules

# Build only this ROS package so colcon does not scan the non-ROS Python packages
# in the main repository.
colcon build --base-paths pkgs/ros_ws/src/holosoma_lidar_cpp_publisher \
  --packages-up-to holosoma_lidar_cpp_publisher
source install/setup.bash
```

If CMake invokes the conda `hsmujoco` Python, install ROS build helpers into that
environment too:

```bash
python -m pip install catkin_pkg
```

Run for the default Livox topic:

```bash
ros2 run holosoma_lidar_cpp_publisher lidar_cpp_publisher --ros-args \
  -p endpoint:=tcp://127.0.0.1:5557 \
  -p topic:=/livox/lidar \
  -p frame_id:=mid360_link_frame \
  -p message_type:=livox_custom
```

Use `message_type:=pointcloud2` when `livox_ros_driver2` is not available or a
standard `PointCloud2` stream is desired.
